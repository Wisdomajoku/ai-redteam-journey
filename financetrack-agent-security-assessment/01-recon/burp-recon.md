# Burp Reconnaissance

## Objective

The objective of this phase was to identify the communication architecture of the FinanceTrack Assistant application, understand how user input reaches the AI agent, and map the exposed attack surface before conducting adversarial testing.

The reconnaissance focused on:

* Identifying application communication channels
* Discovering how chat messages are transmitted
* Observing agent execution flow
* Identifying available tools and actions
* Locating user-controlled input points

## Reconnaissance Environment

Target:

FinanceTrack Assistant

Application Type:

AI agent web application

Local Application URL:

```text
http://localhost:8501
```

Testing Tool:

Burp Suite Community Edition

## Initial HTTP Traffic Analysis

Burp Proxy was first used to inspect HTTP traffic generated by the application.

During initial inspection, repeated Streamlit framework requests were observed:

```http
GET /_stcore/health
GET /_stcore/host-config
```

These requests contained Streamlit initialization and application health information.

No dedicated application chat endpoint was identified through HTTP traffic.

Examples of application endpoints not observed:

```http
POST /chat
POST /api/chat
POST /api/messages
```

Because the HTTP history did not expose the actual chat interaction flow, further analysis was performed through the WebSocket communication channel.

## WebSocket Communication Discovery

The application uses Streamlit WebSocket communication for user interaction.

Observed WebSocket channel:

```text
http://localhost:8501/_stcore/stream
```

The WebSocket channel carried the actual AI agent communication flow, including:

* User prompts
* Agent actions
* Tool execution requests
* Final responses

## Normal Transaction Request Flow

A normal user request was submitted through the application interface:

```text
What are my recent transactions?
```

The following execution flow was observed:

```text
User Input
    |
    v
WebSocket Frame
    |
    v
LangChain Agent Processing
    |
    v
Agent Tool Execution
    |
    v
Final Answer Response
```

## Observed WebSocket Frames

### Frame 1: User Input

The initial user-controlled message was observed in the WebSocket traffic.

Example:

```text
What are my recent transactions?
```

This represents the primary user-controlled input entering the AI agent workflow.

---

### Frame 2: Agent Tool Invocation

The agent selected the `GetCurrentUser` tool.

Observed structure:

```json
{
    "action": "GetCurrentUser",
    "action_input": ""
}
```

Purpose:

Retrieves the current user's identity information.

---

### Frame 3: Agent Tool Invocation

The agent then selected the `GetUserTransactions` tool.

Observed structure:

```json
{
    "action": "GetUserTransactions",
    "action_input": "1"
}
```

Purpose:

Retrieves transaction data associated with the supplied user identifier.

---

### Frame 4: Final Agent Response

The final response was returned using the following structure:

```json
{
    "action": "Final Answer",
    "action_input": "Your recent transactions are displayed..."
}
```

The response contained the transaction information requested by the user.

## Agent Execution Observations

The application exposes tool execution actions through WebSocket communication.

Observed available tools:

| Tool                | Purpose                                                |
| ------------------- | ------------------------------------------------------ |
| GetCurrentUser      | Retrieves current user identity                        |
| GetUserTransactions | Retrieves transaction records based on user identifier |

The internal Thought/Action/Observation reasoning chain was not exposed directly.

The visible communication consisted of:

* User input
* Agent action selection
* Tool inputs
* Final response

## Identified Attack Surface

The primary identified user-controlled input point is:

```text
Chat input field
```

The user message is transmitted through the WebSocket channel and processed by the LangChain agent.

Potential assessment areas identified:

* Direct prompt injection
* Agent instruction manipulation
* Tool misuse
* Unauthorized data retrieval
* Sensitive information disclosure
